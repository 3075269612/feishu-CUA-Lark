# Docs 技能

## 常见入口
- 左侧栏"云文档"图标，通常在消息图标下方
- 顶部搜索框搜索文档
- 快捷键：Ctrl+4 快速切换到云文档

## 创建空白文档流程（Phase 5）

**重要**：此流程通过点击 GUI 元素创建空白文档，使用 VLM + OCR + Accessibility Tree 混合定位。

**目标名称规范**：
- VLM 可以使用**中文或英文**描述 UI 元素，代码会自动映射到标准目标名称。
- 推荐使用自然的中文描述，例如"左侧栏云文档入口"、"新建按钮"等。
- 标准英文名称：`feishu_window`, `cloud_docs_entry`, `new_button`, `doc_type_option`, `new_blank_doc_button`, `doc_title_input`

**生成 step goal 时的关键要求**：
- `target` 字段必须使用**实际的值**，而不是占位符。
- ❌ 错误：`{"target": "title_input"}`
- ✅ 正确：`{"target": "CUA-Dark-Test-Doc"}`（使用 `task.slots.target_doc` 的实际值）

示例 JSON：
```json
[
  {"index": 1, "description": "聚焦飞书窗口", "target": "feishu_window", "expected": "飞书窗口在前台"},
  {"index": 2, "description": "点击左侧栏云文档入口", "target": "cloud_docs_entry", "expected": "进入云文档页面"},
  {"index": 3, "description": "点击新建按钮", "target": "new_button", "expected": "弹出文档类型菜单"},
  {"index": 4, "description": "选择文档类型", "target": "doc_type_option", "expected": "弹出新建文档弹窗"},
  {"index": 5, "description": "点击新建空白文档", "target": "new_blank_doc_button", "expected": "浏览器打开文档编辑页"},
  {"index": 6, "description": "点击标题输入区并输入标题", "target": "doc_title_input", "expected": "标题已输入"}
]
```

流程步骤：

1. **聚焦飞书窗口**
   - 自动检测并调起飞书到前台（启动、还原最小化、置前）。
2. **点击左侧栏"云文档"图标**
   - 使用混合定位系统找到左侧栏中的"云文档"入口并点击。
   - 如果已在云文档页面（页面标题或内容区域包含云文档标识），跳过此步骤。
3. **点击右侧"新建"按钮**
   - 在云文档页面右上角区域定位"新建"按钮并点击。
   - 点击后会弹出下拉菜单，包含"文档"、"电子表格"、"多维表格"、"思维笔记"等选项。
4. **点击弹出菜单中的"文档"选项**
   - 在下拉菜单中定位"文档"选项并点击。
   - 注意不要误点击"电子表格"或"多维表格"。
5. **点击弹窗中的"新建空白文档"**
   - 点击"文档"后会出现一个弹窗，其中包含"新建空白文档"按钮。
   - 点击该按钮后，飞书会自动打开浏览器并跳转到文档编辑页面。
6. **在浏览器中输入标题**
   - 页面加载后会出现"请输入标题"的提示文字。
   - 点击标题输入框区域，然后写入目标标题（如 `CUA-Dark-Test-Doc`）。
7. **完成**：文档已创建，标题已设置。文档会自动保存。

## 常见异常
- 找不到云文档入口：确认飞书窗口完整显示，左侧栏可见；若左侧栏折叠则先展开。
- 新建按钮不可见：确认当前在云文档页面根目录，而非某个子文件夹内。
- 浏览器未打开：确认默认浏览器已安装且可正常启动。
- 标题输入框未找到：等待页面完全加载后再定位；若页面加载超时则返回 blocked。
- 权限不足：某些企业可能限制创建文档的权限。

## 安全边界
- **测试文档标题必须包含 `CUA-Lark` 或 `CUA-Dark` 关键词**，避免污染真实文档库。
- 测试文档应创建在指定的测试文件夹中（如可行）。
- 测试完成后应手动清理创建的文档。
- 不要将测试文档分享给真实用户。

## Dry Run 规则
`--dry-run` 可以执行窗口聚焦、截图、坐标规划和 trace 记录，但不得执行点击或输入。报告最终状态应为 `uncertain`，用于人工确认坐标和动作计划。
